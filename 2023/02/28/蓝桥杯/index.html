使用最小花费爬楼梯

给你一个整数数组 cost ，其中 cost[i] 是从楼梯第 i 个台阶向上爬需要支付的费用。一旦你支付此费用，即可选择向上爬一个或者两个台阶。

你可以选择从下标为 0 或下标为 1 的台阶开始爬楼梯。

请你计算并返回达到楼梯顶部的最低花费。

分析：

首先dp[i]是爬到第i个楼梯的最小花费

初始化dp[0]=0,dp[1]=0

递推公式dp[i]=min{dp[i-1]+cost[i-1],dp[i-2]+cost[i-2]}

整数拆解

343. 整数拆分 - 力扣（Leetcode）

给定一个正整数 n ，将其拆分为 k 个 正整数 的和（ k >= 2 ），并使这些整数的乘积最大化。

返回 你可以获得的最大乘积 。

分析：

这道题也是dp问题，期中比较复杂的是搞清楚dp[i]中i的含义，dp[i]代表数字i的拆分数最大乘积，还要搞清楚递推关系，比如对于任意数字i，有两种方式进行拆解，一种是j(i-j)另一种是j*dp[i-j],取最大值,然后还要分清除递推的方向，是从前到后。外层循环是从前到后，内从循环是进行dp数组的更新

六、回溯法

回溯法并不高效

回溯的本质是穷举，穷举所有可能，有些问题只能暴力解决

回溯法，一般可以解决如下几种问题：

  • 组合问题：N个数里面按一定规则找出k个数的集合
  • 切割问题：一个字符串按一定规则有几种切割方式
  • 子集问题：一个N个数的集合里有多少符合条件的子集
  • 排列问题：N个数按一定规则全排列，有几种排列方式
  • 棋盘问题：N皇后，解数独等等

所有回溯法的问题都可以抽象为树形结构

回溯三部曲：

  • 返回值及参数
  • 回溯函数终止条件
  • 回溯搜索的遍历过程

七、贪心算法

贪心的本质时选择每一阶段的局部最优，从而达到全局最优

那么什么时候用贪心呢？

贪心算法并没有固定的套路，如果局部最优不能推出来全局最优，就要使用动态规划

贪心算法分为下面四步：

  • 将问题分解为若干子问题
  • 找出贪心策略
  • 求每一个子问题的最优解
  • 将局部最优解堆叠成全局最优解

其实只要搞清楚局部最优是什么就好了
