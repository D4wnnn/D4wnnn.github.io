一些基础

  1. 权限修饰符

    四种权限都可以用来修饰类的内部结构：属性 方法 内部类 构造器

    修饰类的话 只能用缺省或者public

  3. 变量

    • 按类型划分:

      基本数据类型（栈里面） 整形：byte（1字节）、short（2字节）、int、long 浮点型：float、double 字符型：char（2字节） 布尔型：boolean（1字节） 引用数据类型（指向堆） String 数组 接口 类

    • 按照声明位置划分:

      局部变量（方法或语句块里面） 成员变量（方法外部或者类的内部）

  4. 输入输出

  5. 抽象类里面的抽象方法只能声明不能定义，只能在子类里面定义

  6. 接口里面所有的方法都是抽象的默认是public abstract

  7. 接口里面所有的变量都是public static final类型，这些标识都可以不写

  8. 抽象类可以有构造方法，而接口不可以

  9. 口中的方法全为抽象方法，而抽象类中也可以有非抽象方法

  10. 类型转换:byte,shotr,int->float->long->double

  11. 重载：必须在同一个类里面 而且参数的类型或者顺序或者数目不同，与修饰符和返回值没关系

  12. 重写:

    必须和父类一摸一样的写（除了权限修饰符） 可以用@Override注解来标识 子类中的访问权限不能低于父类
