2.3 元素显示模式

html元素分为块元素和行内元素两种

常见块元素：div，p，h1，ul，ol，li

特征：

  • 独占一行
  • 宽高、内外边距可以控制
  • 宽度默认是父元素100%
  • 里面可以放块元素或者行内元素
  • ps:文字类的元素里面不能使用块级元素，如p，h1

常见行内元素：a，span

特征：

  • 宽高设置无效
  • 行内元素只能容纳行内元素
  • 特殊：a标签内可以放块级元素

常见行内块元素：img，input，td

特征：

  • ==和相邻的行内块元素在同一行，但是会有缝隙==
  • 默认宽度就是本身内容的宽度
  • 宽高，内外边距都可设置

显示模式的转换：

display:block 转换为块级元素

display:inline 转换为行内元素

display:inline-block 转换为行内块元素

小技巧：当line-height=盒子的高度，文字会垂直居中

四、浮动

传统布局方式

  • 普通流

  • 浮动

    改变元素默认排序方式

  • 定位

网页定位第一准则：多个块级元素纵向排列找标准流 多个块级元素横向排列找浮动

网页定位第二准则：先设置盒子大小 之后设置盒子位置

float创建浮动框 将其移动到一边 直到边缘触碰到块或者另一个浮动框的边缘

浮动特性：

  • 浮动元素会脱离标准流

    浮动的盒子不再保留原来的位置

  • 浮动的元素会一行内显示并且元素顶部对齐

  • 浮动的元素会具有行内块的特性

浮动元素经常和标准流父亲搭配使用：先用标准流的父亲元素排列上下位置，然后内部元素采取浮动排列左右位置

要浮动全浮动

为什么要清除浮动？

由于父级盒子在很多情况下不方便给高度，但是子盒子浮动又不占有位置，最后父盒子高度为0，所以要清除浮动

清除浮动的本质就是清除浮动的影响

若父盒子有高度 则不需要清除浮动

clear:left/right/both

在实际应用中，几乎只用clear:both

清除浮动的方法

  • 闭合标签法：clear:both（新添加的块级元素必须是块级元素）

  • 给父元素添加overflow属性：hidden/auto/scroll 缺点：无法显示溢出的部分

  • ：after伪元素法 也是给父元素添加

  • 双伪元素清除浮动 也是给父元素添加

CSS属性书写顺序

  1. 布局定位属性 建议display第一个写
  2. 自身属性
  3. 文本属性
  4. 其他属性
