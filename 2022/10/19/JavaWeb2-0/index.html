    Servlet中的核心方法?

    • init()

    那么请求是如何执行的呢?

    当后端收到请求时,service方法会自动响应(其实是tomcat容器调用的),然后service方法分析请求方法,根据不同的请求方法执行相应的doXXX方法,默认都是405的报错实现风格,所以要重写doXXX方法;

    那么这些方法什么时候创建,什么时候销毁呢?

    默认情况下,第一次接受请求时,这个Servlet会进行实例化,初始化,然后服务

    从第二次请求开始,每一次都是服务

    容器关闭时,进行销毁

    那么如何调整servlet的启动时间呢?

    在web.xml中设置load-on-startop,数字越小,启动越快,最小为0

    Servlet在容器中为单例的,线程不安全的,故尽量不要在servlet中定义成员变量

    若不得不在serlvet中定义成员变量,那么尽量不要改成员变量的值,并且尽量不要通过成员变量做一些逻辑判断

  9. servlet保存作用域

    保存作用域有四个:

    page request session application

    • page 已经不用了

    • request 一次请求响应范围

      重定向不能共享此作用域

      但是服务器端转发可以

    • session 一次会话范围有效

    • application 一次应用程序有效

      application.setAttribute(...)是公共的,一人设置,所有人共享

Filter

  • 三要素:

    • 拦截
    • 过滤
    • 放行

  • 步骤

    • 实现Filter接口
    • 实现doFilter方法
    • 若满足条件则执行chain.doFilter(request,response)
  • package cn.d4wn.controllers;

import javax.servlet.*;
import javax.servlet.annotation.WebFilter;
import java.io.IOException;

@WebFilter("/index")
public class FilterController implements Filter {
    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
        System.out.println("init");

    }

    @Override
    public void doFilter(ServletRequest req, ServletResponse resp, FilterChain chain) throws IOException, ServletException {
        System.out.println("doFilter");
        int c = Integer.parseInt(req.getParameter("c"));
        System.out.println(c);
        if (c == 1) {
            chain.doFilter(req, resp);
        } else {
            System.out.println("fuckYou");
        }

    }

    @Override
    public void destroy() {
        System.out.println("destroy");
    }
}

    1. 前杠后星 在我们配置了url-pattern为/user/*之后，请求地址只要是/user开头的那么就会被匹配。
    2. 前星后缀 *.png
    3. 不允许这么配置: /*.png

Listener

  • 观察者模式是23种设计模式之一

  • 监听器：专门用于对其他对象身上发生的事件或状态改变进行监听和相应处理的对象，当被监视的对象发生情况时，立即采取相应的行动。 Servlet监听器：Servlet规范中定义的一种特殊类，它用于监听Web应用程序中的ServletContext，HttpSession 和HttpServletRequest等域对象的创建与销毁事件，以及监听这些域对象中的属性发生修改的事件。

  • 监听器列表

    • ServletContextListener监听ServletContext对象的创建与销毁
    • HttpSessionListener监听HttpSession对象的创建与销毁
    • ServletRequestListener监听ServletRequest对象的创建与销毁
    • ServletContextAttributeListener监听ServletContext中属性的创建、修改和销毁
    • HttpSessionAttributeListener监听HttpSession中属性的创建、修改和销毁
    • ServletRequestAttributeListener监听ServletRequest中属性的创建、修改和销毁
    • HttpSessionBindingListener监听某个对象在Session域中的创建与移除
    • HttpSessionActivationListener监听某个对象在Session中的序列化与反序列化。
