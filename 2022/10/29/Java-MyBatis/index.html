创建Mapper接口

相当于以前的dao,但是mapper仅仅是接口,我们不需要提供实现类

创建MyBatis映射文件

两个一致:

  • 映射文件中的namespace和mapper接口的全类名一致
  • 映射文件中的SQL语句id要和mapper接口中的方法名一致

那么如何引入配置文件呢?

在核心配置文件中的mapper标签中设置

MyBatis中的pojo就是orm中的映射的实体类，UserMapper这个接口就是原来的UserDao接口，但是原来的UserDaoImpl不再存在，而是通过UserMapper.xml实现，在这个 xml文件中，通过映射配置，id为接口中的方法名，里面的内容就是SQL语句。

获取参数值

${} 本质是字符串拼接

#{}本质是占位符赋值

如果参数只有一个怎么办呢?

select * from users where username = '#{username}'

若有多个参数呢?

select * from users where username = '#{arh0}' and sex = '#{arg1}'

最终版本如何?

使用注解@Param

User getOne(@Param("username") String username, @Param("password") String password);

@Param("username")就是说select语句中的占位符号username对应形参的username

MyBatis中设置了默认的类型别名

比如Integer就是int
